~!gurí_: Gestor Unificado de formatos para Revistas de Investigación

Descripción de la propuesta

~!gurí_ facilita la generación de documentos finales para revistas científicas a partir de documentos obtenidos en la etapa de ‘corrección de pruebas’. La propuesta busca resolver las dificultades de algunas revistas académicas para la generación de documentos finales en diferentes formatos de forma consistente y sin generar procesos duplicados. Además, toma en cuenta que muchas revistas científicas utilizan como base de sus flujos de trabajos documentos docx.

En términos resumidos, la propuesta propone esquematizar y separar los principales elementos que componen un artículo científico. Para ello, para cada artículo, se debe generar un archivo docx (generado mediante una plantilla y con las citas identificadas con Zotero) y un archivo yaml con los metadatos del artículo. Opcionalmente, puede brindarse otros archivos con información de la taxonomía CRediT, elementos flotantes (figuras y tablas) y apéndices. A partir de estos archivos iniciales se genera un archivo intermedio markdown, que se utiliza para generar los documentos finales (en formato pdf, html y xml). Por su diseño la propuesta permite una fuerte adaptación de la apariencia de los formatos finales, a partir de la utilización de plantillas personalizadas.

La herramienta se basa en el uso de Pandoc para la conversión entre formatos, a lo cual se agrega un conjunto de filtros Lua y plantillas personalizadas, así como un flujo de trabajo, que permite en su conjunto la creación de los documentos finales.

Dependencias de software

Para el uso de esta herramienta se requiere la instalación previa de Pandoc, R y alguna distribución de LaTeX. Si no tiene instalados estos programas deberá hacerlo previamente de las páginas oficiales.

Como se señaló anteriormente, esta herramienta se basa en el uso de Pandoc (versión 3.1.10 o superior) como herramienta de conversión entre formatos.

También es necesario instalar alguna versión de R (recomendable versión 4.3 o superior) y, preferentemente, del entorno de desarrollo RStudio. R es el que es el encargado de coordinar el proceso de conversiones. Para el funcionamiento de ~!gurí_ deberá además instalar los paquetes tidyverse, rmarkdown, readxl, tinytex, crayon. Estos paquetes son instalados automáticamente (si no están instalados aún), mediante la función GURI_install().

Por último, es necesario tener instalada alguna distribución de LaTeX. A las personas con poca experiencia en el uso de LaTeX, recomendamos utilizar la distribución tinytex, por su integración con R y la facilidad para manejar las dependencia de paquetes. LaTeX se utiliza para generar los archivos en formago pdf. Además, es recomendable tener instalados los siguientes paquetes de LaTeX que son utilizados por la pantilla de Pandoc: amsfonts, amsmath, lm, unicode-math, iftex, listings, fancyvrb, longtable, booktabs, graphicx, hyperref, xcolor, soul, geometry, setspace, babel, xeCJK, fontspec, selnolig, mathspec, biblatex, bibtex, biber, upquote, microtype, csquotes, natbib, bookmark, footnotehyper, footnote, xurl, parskip y svg. A su vez, la adaptación de la plantilla utiliza los siguientes paquetes: adjustbox, fontawesome5, caption, ccicons, relsize, truncate, lastpage y koma-script. Si no tiene instalada una distribución de y estos paquetes, pueden ser instalados automáticamente con la función GURI_install(install_tinytex = T).

Licencia

~!gurí_ puede usarse como parte del proceso de producción de revistas de acceso diamante (sin cobro de cargos de procesamiento y sin muros de pago). Más allá de esta restricción, no existe ningún límite salvo la atribución de autoría. En este sentido, se solicita que si su revista utiliza esta herramienta como parte de su proceso editorial agregue el siguiente texto dentro de su página web (habitualmente dentro de la sección de ‘política editorial’) en los diferentes idiomas que utilice en la revista:

Español:
Los documentos finales de esta revista fueron generados utilizando ~!guri_.

English:
The final documents of this journal were generated using ~!guri_.

Português:
Os documentos finais desta revista foram gerados usando ~!guri_.
