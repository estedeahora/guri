Instalación y configuración

Instalación de programas y dependencias

Para utilizar ~!gurí_ deberá previamente haber descargado e instalado Pandoc y R. Además, es conveniente tener instalado el entorno de desarrollo RStudio. Una descripción de la instalación de estos programas está disponible en las páginas oficiales de los mismos.

Una vez que tenga instaladas estos programas, deberá descargar el archivo zip con la última versión del código de ~!gurí_ y colocar los archivos descomprimidos en una carpeta llamada “guri” de su disco duro (no importa la ubicación de la misma). En esta carpeta encontrará el archivo guri.Rproj, una carpeta denominada ./example/ con los archivos de una revista ficticia de ejemplo (utilizada en el documento de Inicio rápido), una carpeta ./files/ con archivos que utiliza ~!gurí_ en el proceso de generación de los documentos (no debería tocar manualmente esta carpeta) y una carpeta ./script/ donde se almacenará el código para generar los documentos finales de sus revistas.

Una vez que se encuentre en la carpeta donde descomprimió los archivos deberá abrir el proyecto de R haciendo doble clic desde el explorador de archivos en el archivo guri.Rproj (puede hacerlo también desde RStudio). Como resultado, se abrirá una sesión de R y RStudio, que dirá “guri” en la parte superior derecha (ver imagen).

A continuación, deberá instalar las librerías de R, así como la distribución de LaTeX (tinytex) y las librerías LaTeX utilizadas. Para ello, cargaremos las funciones utilizadas por ~!gurí_ (guardadas en el archivo /scripts/GURI_00_fx.R) y utilizaremos la función GURI_install() que realiza las instalaciones de forma automática. Para ello ejecutar en la consola de R el siguiente comando (puede tardar algunos minutos):

source("scripts/GURI_00_fx.R", encoding = "UTF-8")
GURI_install()

Para verificar que esté instalado correctamente puede ejecutar el ejemplo (ver Inicio rápido).

Creación de una revista

~!gurí_ está pensado para que pueda administrar una o más revistas de forma conjunta. Para ello, se genera un directorio para cada una de las revistas y un script personalizado para la creación de los documentos de su revista (en la carpeta /scripts/). Para automatizar la creación de su revista puede utilizar el comando GURI_make_journal que generará una carpeta y el conjunto de archivos predeterminados para la configuración de la misma. Para ello, debe suministrar un “nombre abreviado” para su revista (reemplazando JUOURNAL en el código), el cual será utilizado para generar las carpetas y para uso interno del programa. Es recomendable no utilizar espacios ni símbolos en el nombre de la revista para evitar errores.

source("scripts/GURI_fx.R", encoding = "UTF-8")
GURI_make_journal("JUOURNAL")

Una vez que finalice, se abrirán dos archivos. En primer lugar, un archivo con el nombre GURI_[JUOURNAL].R (ubicado en la carpeta /scripts/). Este archivo contiene el script para generar los documentos de su revista y lo utilizaremos más adelante. En segundo lugar, el archivo ./JUOURNAL/_journal.yaml, que contiene los datos básicos de la revista. Antes de cerrar este último archivo debe completar los datos de su revista, que son los que luego utilizará para generar los archivos finales. En caso de ser necesario, puede modificar y ampliar los datos más adelante, aunque recomendamos completar la mayor cantidad de datos posibles antes de comenzar a generar sus documentos. Si algún campo no corresponde o no será usado por su revista, utilice # al inicio de la línea y el campo será omitido. Es importante que no modifique el tabulado a la izquierda dentro de este archivo, ya que el mismo se utiliza para identificar el anidamiento de los mismos.

Configuraciones globales de la revista. La carpeta _config

Cada revista puede tener, opcionalmente, otros archivos que permiten personalizar la salida de cada uno de los formatos. Estos archivos se encuentran en la carpeta /_config/ dentro del directorio de su revista (./JUOURNAL/_config). En caso de que esta carpeta no se genere se utilizarán las plantillas por defecto. De forma predeterminada, el comando GURI_make_journal genera esta carpeta con una copia de los archivos utilizados por defecto. Describimos a continuación los archivos que puede utilizar en esta carpeta.

Si proporciona en la carpeta /_config/ un archivo .csl (Citation Style Language), el mismo será usado por defecto para generar las referencias. Puede descargar estos archivos de Zotero Style Repository. El estilo de citado será utilizado para todos los formatos de salida de forma consistente. En caso de que no se proporcione ningún estilo de citado, se utilizará la configuración por defecto de Pandoc.

Para modificar la salida en formato pdf, opcionalmente se puede proporcionar en la carpeta de configuración un archivo latex_metadata.yaml. Este archivo permite definir, entre otros elementos, características de las fuentes y de la composición de la página. Si es necesario una mayor personalización de la salida pdf, es posible utilizar los archivo template.latex y custom_journal.sty, que se utilizarán como plantilla para generar el archivo tex (LaTeX). Configuración avanzada con filtros personalizados

Es posible también agregar filtros personalizados que intervengan en el AST de pandoc (ver Pandoc). Sólo se aceptan filtros escritos en lua. Los filtros se aplican luego de que todos los filtros predefinidos por guri se apliquen. Para agregar un filtro personalizado debe colocar un archivo .lua en la carpeta. El mismo debe tener como prefijo el tipo de salida (md, html, jats, tex), un número (que indica el orden en que se aplicará el filtro), un nombre (ej: “md_1_filter-name.lua”).
